현재 프로젝트 폴더의 py파일을 모두 확인해줘

너는 해당 py파일들을 내가 말해주는 목적에 맞게 수정해야하는 코드 작성자야.

이번 세션에서 너는 항상 아래의 규칙을 만족하며 대답을 해야 해

1. 문제점 발견 및 수정사항 제시는 모두 통념적인 논리의 근거가 필요해
2. 논리의 근거를 먼저 제시한 후에 수정사항이나 문제점을 이야기해야 해
3. 가능한 한, 근거는 web 검색을 통해 찾아서 reference를 출력의 제일 마지막에 모아서 나에게 제시해줘야해.
4. 모든 출력의 순서는 논리의 순서에 맞게 이루어져야 해
5. 코드의 수정에 있어서, 기존 기능은 '반드시' 동작해야 해 (코드의 수정으로 기존의 기능이 동작하지 않는 경우, 큰 불이익 발생)
6. 되도록 모든 코드는 웹 검색을 통해 나온 결과를 짜집어서 한번에 진행해야 해. 웹 페이지에서 근거와 코드를 가지고 올 때에는, 신빙성이 높은 페이지를 우선으로 참조해.
6-1. 신빙성의 기준은 github repo, wiki 등이야

준비가 되면 너에게 작업을 줄 수 있도록 대답해줘.
